# Roadmap реализации: Pipelines + Pre/Post Processing (по спекам)

Эта папка — **план постепенной реализации** функционала из спеки `docs/pipelines-and-processing-spec/` (v0.1), с движением **от простого к сложному**.

- Каноническая спека: `docs/pipelines-and-processing-spec/map.md`
- Главный трекер статуса: `docs/pipelines-and-processing-implementation/00-status.md`

## Как пользоваться

- Все работы разбиты на **фазы** (файлы `10-*.md`, `20-*.md`, …).
- В каждой фазе есть список задач и критерии готовности.
- **Статус ведём только в одном месте**: `00-status.md` (чекбоксы + пометки “done/in-progress/blocked”).

## Принципы (переиспользуем из спеки)

- Backend — **source of truth** для “что пошло в LLM и почему”.
- В v1 не переписываем chat-core: `llm` шаг — это **boundary** существующей генерации; добавляем обвязку `pre/post`.
- По умолчанию: prompt-time мутации разрешены, “исторические трансформации” — **отдельная история (v2+/opt-in)**.
