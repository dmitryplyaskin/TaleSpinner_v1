# TaleSpinner vs SillyTavern

Срез сравнения: **12 февраля 2026**.

## Что я сравнивал

Сравнение сделано по тому, как написана и структурирована пользовательская документация в `docs.sillytavern.app`, и как это отражается в реальном onboarding.

SillyTavern docs обычно дают:

- четкий task-first формат,
- пошаговые сценарии,
- понятные точки входа для новичка,
- меньше внутренней терминологии в first-run документах.

Этот User Guide TaleSpinner теперь выровнен под такой же подход.

## Практическая разница продуктов

| Критерий | TaleSpinner | SillyTavern |
| --- | --- | --- |
| Pipeline вокруг генерации | Встроенные Operations (hooks, dependsOn, required, node-editor) | Обычно достигается комбинацией расширений и ручной конфигурации |
| World Info orchestration | Multi-scope binding (`global/chat/entity_profile/persona`) | Сильная экосистема, но сценарии часто собираются более вручную |
| Миграция существующих данных | Поддержка импортов `st_native`, `character_book`, `agnai`, `risu`, `novel` | Нативная среда для ST-форматов |
| Подход к UX | Более связанный end-to-end workflow из коробки | Максимальная гибкость и расширяемость |

## Когда выбирать TaleSpinner

Выигрыш обычно заметен, если вам нужно:

- меньше ручной склейки между функциями,
- повторяемый pipeline генерации,
- управляемый рост сложных сценариев (World Info + Operations + presets).

## Когда выбирать SillyTavern

SillyTavern часто лучше, если вам критично:

- максимально широкий каталог community-расширений,
- deep кастомизация под личный workflow,
- быстрые эксперименты на базе уже готовой ST-экосистемы.

## Итого

Оба инструмента сильные, но под разный стиль работы:

- SillyTavern - про максимальную гибкость и экосистему,
- TaleSpinner - про более интегрированный и воспроизводимый рабочий процесс.
